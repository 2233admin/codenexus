**Wall-clock budget for synthetic tests verified at plan time:**

R4.b synthetic budget: `MAX_CONSECUTIVE_FAIL_DEFAULT = 5` (server.rs); each
`embed()` call has `MAX_ATTEMPTS = 5` × `BASE_DELAY_MS = 250` exponential
backoff = ~7.75s wall-clock per failed embed. With `CODENEXUS_EMBED_FAIL=always`,
each embed call burns the full retry budget then fails. After 5 consecutive
fails, server bails. Total budget: 5 fails × 7.75s = ~39s. Test asserts
< 60s as headroom.

R5.b synthetic budget: `QUERY_MAX_ATTEMPTS = 2` × `QUERY_DELAY_MS = 250` flat
= 250ms sleep + 2 × ~50ms processing ≈ 350ms. Test asserts < 1.0s as headroom.
v1's claim of <1s budget for query failures is preserved.

  exit 52
fi
echo "PASS: R5.b — query failure in ${ELAPSED_MS}ms (< 1000ms budget)" | tee -a "$LOG"

echo "==[ ALL E2E + SYNTHETIC PHASES PASSED ]==" | tee -a "$LOG"
