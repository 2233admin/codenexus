# Phase 4 (Group 2 Entry): Multi-Language Tree-Sitter -- Specification

**Created:** 2026-04-28
**Ambiguity score:** 0.11 (gate: <= 0.20) -- PASS

**Phase number:** 4 (group 2 entry slice; subsequent group-2 slices for Rust / Java / C++ + group 2.5 graph_build per-language each get their own SPECs)

## Goal

Extend `experiments/poc-retrieval/src/parser.rs` to support **Go** as the first multi-language addition after the 04-08 spike (TypeScript + Python entry), normalize `Symbol.kind` into a dual-field representation (`kind_raw` + `kind_norm`) so A2A clients consume a stable cross-language taxonomy, and lock all `tree-sitter*` grammar crates to exact versions in `Cargo.toml` so future ABI churn (like the 0.22 -> 0.25 churn that triggered the spike) is bounded by explicit upgrade decisions instead of cargo-default caret drift.

## Background

After the 04-08 spike (commits `84f1e97` + `c9d31a5` + `3585026`), `parser.rs` already contains the architectural primitives needed to scale to N languages: `Language` enum (currently `Typescript` + `Python`), `LangCtx` struct (compiled once per language), `detect_language` from file extension, and `parse_repo` dispatch via a per-file extension match. Adding Go costs ~10 lines per the spike's evidence-backed recipe (`Cargo.toml +1`, `Language::Go` enum variant, `QUERY_SRC_GO` const, `detect_language` arm, `LangCtx::new` line, dispatch arm).

Two issues remain unsolved by the spike that downstream slices need locked first:

1. **`Symbol.kind` is the raw `bn.kind()` string.** TS yields `function_declaration`, Python yields `function_definition`, Go will yield `function_declaration` (same node name as TS, different grammar). A2A clients (`get_symbol`) consume this opaquely today. Without normalization, every new language permanently increases client-side cross-grammar mapping burden, and the same name across grammars (e.g. TS `function_declaration` vs Go `function_declaration`) silently equates to two different things in different files. Backwards-compat cost of changing API shape later is high.

2. **`Cargo.toml` uses cargo-default caret semver.** The 0.22 -> 0.25 churn forced spike step A (commit `84f1e97`) to rewrite 7 call sites. Group 2 will add Go now and Rust / Java / C++ later -- each new grammar crate increases the future ABI-drift surface. Pinning to exact versions converts every future `tree-sitter*` upgrade into an explicit project-controlled decision, at the cost of manually tracking RustSec advisories.

`graph_build.rs` (Calls + Imports + Implements + Extends edges) is hardcoded to TypeScript grammar node names. Extending it to Python / Go / Rust requires per-language edge resolvers that are non-trivial (Python imports use `import x` and `from y import z` syntax different from TS `import { x } from 'y'`; Go has no `interface` keyword in the TS sense; etc.). The spike notes (`04-08-SPIKE-NOTES.md` "What's still open" #3) recommend separating this concern. This SPEC accepts that recommendation: `graph_build` extension is **out of scope** here and gets its own slice (group 2.5).

The cross-language eval set (verifying NL retrieval works *across* languages, not just *within* TS) is also explicitly out of scope here and deferred to a wrap-up slice after group 2 finishes adding all target languages. Adding it incrementally per-language risks cherry-picking queries that work; building it once at the end forces unified concept design.

### R2: Symbol.kind dual-field taxonomy

Replace single `Symbol.kind: String` with `Symbol.kind_raw: String` (existing semantics, raw `bn.kind()`) plus `Symbol.kind_norm: SymbolKind` enum where `SymbolKind` is one of `{Function, Class, Method, Variable, Type, Module}`. Storage schema migrates additively (ADD COLUMN + backfill from kind_raw via lookup table); A2A `get_symbol` response includes both fields; existing clients consuming `kind` see no breakage IF a backwards-compat alias is preserved (planner decides shape; recommended to keep `kind` field as alias of `kind_raw` for one slice grace period).

- **Current**: `Symbol { kind: String, name, path, start_line, end_line, snippet }`. `storage::Store` schema has single `kind` column. A2A `get_symbol` returns `kind` only. Cross-grammar same-name ambiguity (TS `function_declaration` vs Go `function_declaration`) silently equates to one cross-language string.
- **Target**: `Symbol { kind_raw: String, kind_norm: SymbolKind, name, path, start_line, end_line, snippet }` (rename existing `kind` -> `kind_raw`); `SymbolKind` enum lives in `parser.rs` or a new `kind.rs`; lookup table maps grammar-specific raw node names to normalized variants; SQLite schema adds `kind_norm` column with backfill UPDATE statement; A2A response shape adds `kind_norm` field.
- **Acceptance** (all 4 required):
  - (a) `grep -E 'pub enum SymbolKind' experiments/poc-retrieval/src/` -> 1 hit, with all 6 variants (`Function|Class|Method|Variable|Type|Module`)
  - (b) `grep -E 'kind_raw|kind_norm' experiments/poc-retrieval/src/{parser,storage,server,search}.rs` -> hits in all 4 files (parser produces, storage persists, server/search consume)
  - (c) Migration test: `migrations/0NN_add_kind_norm.sql` exists; running on the existing 2116-symbol poc.db backfills `kind_norm` for >=99% of rows (the 1% slack covers any unmappable raw names that legitimately fall through to a `kind_norm = NULL` fallback)
  - (d) A2A response inspection: `curl POST /tasks/send` with a `GetSymbol` op returns JSON containing both `kind_raw` and `kind_norm` keys

### R4: Grammar crate exact-lock policy

Lock all `tree-sitter*` deps **and `streaming-iterator`** in `experiments/poc-retrieval/Cargo.toml` to exact versions (`= "x.y.z"`) so cargo's default caret semver cannot pull patch upgrades that break our query strings or call-site code. `streaming-iterator` is included because it is a tree-sitter ABI bystander -- the spike step A (commit `84f1e97`) added it precisely because tree-sitter 0.25 made `cursor.matches()` return a `StreamingIterator` not a stdlib `Iterator`. An upstream `streaming-iterator` major-version break would force the same 7-call-site rewrite pattern as the original tree-sitter churn, so it gets the same lock discipline.

- **Current**: `Cargo.toml` uses cargo-default caret semver (e.g. `tree-sitter = "0.25"` resolves to `^0.25` -> any `0.25.x` patch acceptable; same for `streaming-iterator = "0.1"`). 0.22 -> 0.25 churn already showed grammar crate ABI instability is real, and `streaming-iterator` was the lever that propagated the churn into our parser.rs / graph_build.rs call sites.
- **Target**: All `tree-sitter*` entries (`tree-sitter`, `tree-sitter-typescript`, `tree-sitter-python`, `tree-sitter-go`, plus future grammars) AND `streaming-iterator` use strict-equality (`= "x.y.z"`). New ARCH section or §9.X history row records the exact-lock policy + rationale (including the streaming-iterator inclusion reason).
- **Acceptance** (all 4 required):
  - (a) `grep -E '^tree-sitter[a-z-]*\s*=\s*"' experiments/poc-retrieval/Cargo.toml | grep -v '"='` -> 0 hits (every match must use `="`)
  - (b) `grep -E 'tree-sitter[a-z-]*\s*=\s*\{' experiments/poc-retrieval/Cargo.toml | grep 'version\s*=\s*"[^=]'` -> 0 hits (table-form deps must also use exact)
  - (c) `grep -E '^streaming-iterator\s*=\s*"=' experiments/poc-retrieval/Cargo.toml` -> 1 hit (streaming-iterator strict-locked)
  - (d) `docs/ARCHITECTURE.md` has a new section or §9.X history row documenting the policy with the locked versions (e.g. "tree-sitter = 0.25.X + streaming-iterator = 0.1.X locked 2026-04-28; upgrade requires explicit ARCH update and test re-run")

## Constraints

- **REQ-10 no-regression**: B1-B7 mean precision_at_5 must stay in `[65.9%, 69.9%]` (= 67.9% +/- 2pp). Post-slice run is mandatory; if mean drops below 65.9%, the slice is not done -- root-cause investigation required (likely candidates: kind_norm UPDATE queries broken, or schema migration corrupted poc.db, or storage::Store query plan changed).
- **target/test-tmp/{uid}/ for parser fixtures**: Windows %TEMP% lives under hidden `AppData\Local\Temp` which `ignore::WalkBuilder` filters out by default; spike's documented workaround applies to the new Go fixture too.
- **Exact-lock scope**: applies to `tree-sitter*` family AND `streaming-iterator` (tree-sitter ABI bystander -- see R4 rationale). Other deps (`anyhow`, `ignore`, `serde`, etc.) retain their existing semver. The lock-list is intentionally narrow; expanding it requires a separate decision recorded in ARCH.
- **Apache 2.0 license compatibility**: tree-sitter-go is MIT-licensed -- compatible with Apache 2.0. No GPL/AGPL grammar crates allowed (per PROJECT.md Out of Scope).
- **Single-fat-binary distribution invariant preserved**: this slice does not change packaging; Go grammar adds ~3-5 MB to release binary (tree-sitter-* parsers are typically 100-500 KB compiled), within ARCH §3 binary-size budget.
- **Schema migration must not corrupt existing poc.db**: backfill via lookup table; if any kind_raw value has no mapping (unlikely if lookup table covers TS + Python + Go grammar nodes), `kind_norm` left NULL with a warning logged. Migration test in R2 acceptance covers this.
- **Test thread serialization**: `cargo test --test-threads=1` remains the official test invocation pattern (P2 deferred fix per `feedback-graduated.md` mention; CODENEXUS_EMBED_FAIL race not solved this slice).

## Framework decisions inherited by group 2 follow-up slices

Future group 2 slices (04-10 if it exists / 04-11 Rust / 04-12 Java/C++ / 04-13 graph 2.5 / 04-14 cross-lang eval wrap-up) inherit this SPEC's framework decisions and write only a **thin acceptance addendum** (~30-50 lines) listing language-specific deltas. The framework decisions inherited verbatim:

1. **`Symbol.kind_raw` + `Symbol.kind_norm` dual-field taxonomy** (R2). New languages contribute new `kind_raw` strings and a lookup-table extension for `kind_norm`. The 6-variant `SymbolKind` enum (`Function|Class|Method|Variable|Type|Module`) does NOT extend by default; if a language has a concept not in the set (e.g. Rust `trait` -> `Method`? `Type`? new variant?), that decision is a per-slice question the addendum answers, but the default is "fit into existing 6 variants, do not extend".
2. **Per-language fixture pattern** (R3). Single-language fixture (>=3 symbols of distinct kinds) + cross-lang mixed fixture (the new language + at least one existing language) + REQ-10 B1-B7 mean precision_at_5 +/- 2pp no-regression gate. This is the "simple" verification shape locked here; future slices that want grammar-coverage tables (the verbose option Curry rejected this round) need to argue for it explicitly.
3. **Exact-lock policy** (R4). Every new grammar crate (`tree-sitter-rust`, `tree-sitter-java`, `tree-sitter-cpp`, etc.) lands in `Cargo.toml` with strict-equality semver (`= "x.y.z"`). The lock-list now includes `streaming-iterator` (tree-sitter ABI bystander); future bystanders that show ABI propagation get added with a one-line ARCH justification.
4. **Co-location boundary** (R5). Group 2 follow-up slices stay scoped to symbol extraction; `graph_build.rs` extension belongs in group 2.5 (slice 04-13); cross-lang eval set belongs in group 2 wrap-up (slice 04-14). Touching these from a non-graph-2.5 / non-04-14 slice is review-blocked per `feedback chestertons-fence-after-fix` discipline.
5. **`target/test-tmp/{uid}/` for parser fixtures**. Windows hidden-attribute pitfall is documented and the workaround is the project standard. Future slices use the same pattern; do not re-introduce `std::env::temp_dir()`.
6. **REQ-10 no-regression gate calibration** (Constraints + R3). Baseline 67.9% from Phase 03.6 byte-identical run is the anchor. Each new slice adds its REQ-10 result to the eval/ directory; baseline does NOT shift (each slice measures against the original, not the prior slice's output) until a deliberate baseline-rebase decision is made in a future SPEC.
