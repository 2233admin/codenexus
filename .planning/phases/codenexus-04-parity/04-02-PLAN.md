<objective>
Apply Phase 3.5b's `consecutive_fails` resilience pattern from `main.rs:Index`
to two same-crate call sites that currently lack it:
- R4: `server.rs` A2A `IndexRepo` handler — long loop over symbols can hit
  silent partial state on cluster failure; needs counter + structured
  failed-task abort.
- R5: `search.rs:31` Query path — currently uses the shared 5-attempt retry
  wrapper (~7.75s sleep budget) which punishes interactive search UX. Add
  `embed_query()` method on `Embedder` with ≤2 attempts ≤500ms total budget.

Purpose: Eliminate two silent-failure / slow-failure modes in long-running
server (R4) and interactive Query path (R5). Mechanical patch only —
NO `EmbedError` enum (Q5=B locked), NO behavior change for Index callers
that use the shared `embed()` wrapper.

Output:
- New `embed_query(&self, text: &str) -> Result<Vec<f32>>` method on
  `Embedder` with `QUERY_MAX_ATTEMPTS=2` and `QUERY_DELAY_MS=250` (NO
  exponential backoff).
- `search.rs:31` switched from `embedder.embed(query, Role::Query)` to
  `embedder.embed_query(query)`.
- `server.rs:dispatch::IndexRepo` handler gains a `consecutive_fails`
  counter loop that bails into A2A `OperationResponse::IndexRepo` failure
  via Result-error propagation when threshold reached.
- A2A `OperationRequest::IndexRepo` envelope extended with optional
  `metadata.max_consecutive_fail` source (per A2A 1.0 free-form
  `metadata` field, verified by Plan-Time Gate G-04).
- The shared 5-attempt `embed()` wrapper at `embedder.rs:84-101` is left
  byte-identical (Index callers in `main.rs` and `server.rs` continue using
  it; the counter sits in caller scope per ARCH §9.9 layer ownership).
</objective>

   ```rust
   /// surfaces as Err in <1s, NOT 7.75s like the shared Index wrapper.
   ///
   /// Per ARCH §9.9 D-W9: caller-policy split — Query callers (interactive
   /// UX) need fast-fail, Index callers (long indexing loops) need the
   /// 5-attempt wrapper. Same `embed_once` primitive, two retry policies.
   pub fn embed_query(&self, text: &str) -> Result<Vec<f32>> {
       const QUERY_MAX_ATTEMPTS: u32 = 2;
       const QUERY_DELAY_MS: u64 = 250;
       let mut last_err: Option<anyhow::Error> = None;
       for attempt in 0..QUERY_MAX_ATTEMPTS {
           match self.embed_once(text, Role::Query) {
               Ok(v) => return Ok(v),
               Err(e) => {
                   last_err = Some(e);
                   if attempt + 1 < QUERY_MAX_ATTEMPTS {
                       std::thread::sleep(std::time::Duration::from_millis(QUERY_DELAY_MS));
                   }
               }
           }
       }
       Err(last_err.unwrap())
   }
   ```
