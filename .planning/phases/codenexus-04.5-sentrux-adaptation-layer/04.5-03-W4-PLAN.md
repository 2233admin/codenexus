    `(call_expression function: (identifier) @callee)`
    `(call_expression function: (member_expression
       object: (identifier) @ns_obj
       property: (property_identifier) @callee))`
    + same for new_expression
  CallsWalker's job: produce equivalent (callee_name, ns_obj, row)
  triples by walking the AST instead of hardcoding the query. Two
  approaches:
  (a) DYNAMIC: build the tree-sitter Query at runtime from the
      [calls] config. Hardest. Most flexible.
  (b) HYBRID: keep a parameterized tree-sitter Query template that
      uses node-kind names from config; instantiate per repo prepare.
  (c) MANUAL WALK: walk the tree-sitter syntax tree by node kind
      directly (no Query object). For each call-expression-kind node,
      inspect the function/constructor field. Method-call detection
      uses node.kind() == method_call_pattern (e.g. "member_expression").

    debuggable.
  - Path forward to Python: a Python walker walks node kinds
    `call`, `attribute` -- different shape, but same pattern.
    Not a "one query string for all" abstraction.

```rust
//! CallsWalker -- native TS calls extractor (Phase 04.5-03 W4).
//!
//! Consumes `[calls]` section of `parser/plugins/typescript/plugin.toml`
//! (L3.a per PRE-PLAN-NOTES). Walks the tree-sitter AST manually (per
//! W4 plan_time_verification decision: option (c) manual walk over
//! option (a) dynamic Query construction).
//!
//! Behavior preservation: emits FoundEdges equivalent to the pre-04.5-03
//! Q_CALLS tree-sitter query (graph_build.rs lines 38-53):
//! - `(call_expression function: (identifier) @callee)`
//! - `(call_expression function: (member_expression
//!     object: (identifier) @ns_obj
//!     property: (property_identifier) @callee))`
//! - same for `new_expression`.

use codenexus_types::Symbol as _;  // unused but reserved
use crate::repo_ctx::RepoCtx;
use crate::{EdgeKind, FoundEdge};

pub struct CallsWalker;

impl CallsWalker {
    /// Extract Calls FoundEdges from a file's source. Reads the
    /// LanguageSemantics's [calls] block to know which node kinds count
    /// as call sites + which member-call shape to match.
    pub fn extract(file: &str, source: &str, repo_ctx: &RepoCtx) -> Vec<FoundEdge> {
        let calls = match get_calls_block(repo_ctx) {
            Some(c) => c,
            None => return Vec::new(),  // no [calls] config -> no extraction
        };

        let mut parser = match build_parser(repo_ctx) {
            Some(p) => p,
            None => return Vec::new(),
        };

        let tree = match parser.parse(source, None) {
            Some(t) => t,
            None => return Vec::new(),
        };

        let mut out = Vec::new();
        walk_node(
            tree.root_node(),
            source.as_bytes(),
            file,
            calls,
            &mut out,
        );
        out
    }
}

/// Get a reference to the &CallsBlock from the repo's LanguageSemantics.
/// Returns None if the [calls] block is empty (no call_expression_kinds).
fn get_calls_block<'a>(repo_ctx: &'a RepoCtx) -> Option<&'a crate::plugin::manifest::CallsBlock> {
    // EXECUTOR: adapt to actual LanguageSemantics field shape from W1.
    // Likely path: repo_ctx.language_semantics.manifest.calls
    // OR: repo_ctx.language_semantics.calls (if W1 flattened it).
    let calls = &repo_ctx.language_semantics.manifest.calls;
    if calls.call_expression_kinds.is_empty() {
        None
    } else {
        Some(calls)
    }
}

fn build_parser(_repo_ctx: &RepoCtx) -> Option<tree_sitter::Parser> {
    // For W4 (TS only), hardcode the tree-sitter-typescript language.
    // 04.5-07 generalizes this to dispatch by language id from
    // repo_ctx.language_semantics.manifest.language.id.
    let mut parser = tree_sitter::Parser::new();
    let lang: tree_sitter::Language = tree_sitter_typescript::LANGUAGE_TYPESCRIPT.into();
    parser.set_language(&lang).ok()?;
    Some(parser)
}

/// Recursively walk the tree-sitter Node tree. For each node whose kind() is
/// in call_expression_kinds, inspect its function/constructor field and emit
/// a FoundEdge.
fn walk_node(
    node: tree_sitter::Node,
    source: &[u8],
    file: &str,
    calls: &crate::plugin::manifest::CallsBlock,
    out: &mut Vec<FoundEdge>,
) {
    let kind = node.kind();

    // Match call-expression-kind nodes (call_expression / new_expression for TS).
    if calls.call_expression_kinds.iter().any(|k| k == kind) {
        // The function-call target lives in the `function` (call_expression)
        // or `constructor` (new_expression) named-child. Tree-sitter exposes
        // both via field_id "function" / "constructor" -- TS grammar uses
        // "function" for both, so probe by name.
        let target_node = node
            .child_by_field_name("function")
            .or_else(|| node.child_by_field_name("constructor"));
        if let Some(t) = target_node {
            if let Some(fe) = extract_from_target(t, source, file, calls) {
                out.push(fe);
            }
        }
    }

    // Recurse into children.
    let mut walker = node.walk();
    for child in node.named_children(&mut walker) {
        walk_node(child, source, file, calls, out);
    }
}

/// Extract a FoundEdge from a function-call target node. Returns None if the
/// target shape doesn't match the supported patterns:
/// - `identifier` -> simple call `foo()`, ns_obj=None
/// - `member_expression` (kind == calls.method_call_pattern) -> `obj.fn()`,
///   ns_obj=Some(object), to_raw=property
fn extract_from_target(
    target: tree_sitter::Node,
    source: &[u8],
    file: &str,
    calls: &crate::plugin::manifest::CallsBlock,
) -> Option<FoundEdge> {
    let row = target.start_position().row + 1;
    let kind = target.kind();

    if kind == "identifier" {
        let to_raw = std::str::from_utf8(&source[target.byte_range()]).ok()?.to_string();
        return Some(FoundEdge {
            from_file: file.to_string(),
            from_name: String::new(),  // enclosing-symbol resolution happens in graph_build.rs::enclosing_symbol
            to_raw,
            ns_obj: None,
            kind: EdgeKind::Calls,
            row,
        });
    }

    if kind == calls.method_call_pattern {
        // member_expression: object: ..., property: property_identifier (TS).
        // Object may be identifier (-> ns_obj is the object name) or another
        // member_expression (-> we ignore deep chains today; pre-04.5-03 also
        // ignored them per Q_CALLS shape line 44-45 + 47-49 which only matched
        // `object: (identifier)`).
        let object_node = target.child_by_field_name("object")?;
        let property_node = target.child_by_field_name("property")?;

        let to_raw = std::str::from_utf8(&source[property_node.byte_range()]).ok()?.to_string();
        let ns_obj = if object_node.kind() == "identifier" {
            std::str::from_utf8(&source[object_node.byte_range()]).ok().map(String::from)
        } else {
            // Pre-04.5-03 Q_CALLS had a third alternate (member_expression
            // without object capture, line 44-45 + 50-51) for cases where
            // the object is NOT a simple identifier. To preserve that
            // behavior, emit the FoundEdge with ns_obj=None when object
            // is non-identifier.
            None
        };

        return Some(FoundEdge {
            from_file: file.to_string(),
            from_name: String::new(),
            to_raw,
            ns_obj,
            kind: EdgeKind::Calls,
            row,
        });
    }

    None
}
```
