**UQ1 RESOLUTION (locked at W1 plan-time, 2026-05-02; preempts cargo dep cycle that would otherwise surface at W3):** option (a) -- extract a `codenexus-types` leaf sub-crate consumed by both `core` and `parser`. The cycle was foreseeable: W3 will replace `resolve_with_*` with `CallResolver::resolve(found, aliases, symbols)`, requiring `core::graph_build` to call into `parser::call_resolver`. That means `core` depends on `parser`. If `parser` ALSO depended on `core` (for `AliasDecl` + `Symbol`), cargo would error on circular path-dep. Resolution: `codenexus-types` (leaf, dependency-free) holds `Symbol` + `AliasDecl` + `EdgeKind` + `ResolvedEdge` + `FoundEdge` + `ResolutionMethod`. Both `core` and `parser` depend on `codenexus-types` only (no inverse dep arrow). W3 then adds `core -> parser` cleanly with no cycle.

Therefore W1 ships `parser` as a workspace member that depends on `codenexus-types` (NOT `codenexus-core`). Symbol moves out of `core::parser::Symbol` and into `codenexus-types::Symbol` as PART OF W1 (Task 0). AliasDecl was placed in W0 in `core/src/types.rs` -- W1 also migrates it to `codenexus-types` and leaves a thin re-export shim in `core/src/types.rs` for source-compat during transition. Document this resolution in the new `parser/src/lib.rs` module-level doc.

Design + ship the `[calls]` TOML schema (L3.a critical addition). Sentrux's `[semantics]` is import-centric; calls extraction needs new fields per UQ3:
- `call_expression_kinds: [string]` -- tree-sitter node kinds that indicate a call (TS: `["call_expression", "new_expression"]`)
- `method_call_pattern: string` -- AST shape for `obj.method()` (concrete: a tree-sitter sub-query string OR a node-kind name; default for TS: `"member_expression"` matching `function: (member_expression ...)`)
- `dynamic_dispatch: bool` -- whether resolver should attempt namespace-alias lookup before falling back to global-unique (TS: true; future Python: TBD)

Write `parser/plugins/typescript/plugin.toml` matching sentrux's TS config + the new `[calls]` section. The plugin.toml ships in this plan but the calls-extractor that CONSUMES the [calls] section lands in W4 (not W1).

This plan is the **largest of W0-W5** (~2200 LoC of verbatim port + type adaptation). Type adaptation pattern: replace sentrux's `Symbol`, `EntryPoint`, `LanguageProfile` references with codenexus equivalents -- mostly mechanical search/replace, with three known re-targets enumerated in plan_time_verification below.

Purpose: get the entire sentrux plugin/parser framework compiling under `cargo build -p parser` BEFORE wave W3 rewires graph_build.rs to call into it. Decoupling means a W1 type-adaptation mistake is caught by `cargo build` not by REQ-10 regression.

//! codenexus-types -- leaf type-shared crate (UQ1 resolution per
//! Phase 04.5-03 W1 plan-time decision, 2026-05-02).
//!
//! Both `codenexus-core` and `codenexus-parser` depend on this crate.
//! Adding deps here would re-introduce a cycle -- keep this crate
//! dependency-free.
//!
//! Contains the locked vocabulary primitives per /CONTEXT.md
//! (commit a0fa4d8). Adding new types here requires a CONTEXT.md update
//! first.

#[derive(Debug, Clone, PartialEq, Eq)]
pub struct Symbol {
    pub kind: String,
    pub name: String,
    pub path: String,
    pub start_line: usize,
    pub end_line: usize,
    pub snippet: String,
}

/// File-scoped name aliasing from an import declaration. See CONTEXT.md
/// lines 45-52 for the four-import-variants contract (named / renamed /
/// namespace / default).
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct AliasDecl {
    pub from_file: String,
    pub alias: String,
    pub target_file: String,
    pub target_member: Option<String>,
}

// Forward-declared placeholders for W2 / W3 / W4. They live here (not
// in core or parser) because the call-graph crosses both crates and
// any consumer of these types must avoid creating a cycle.
//
// W2 fleshes ResolutionMethod (call_resolver land).
// W3 fleshes ResolvedEdge (storage projection land).
// W4 fleshes FoundEdge + EdgeKind (calls_walker land).
//
// Keeping them as W1-stubs vs deferring entirely: stubs let downstream
// waves `use codenexus_types::ResolutionMethod` from day one without a
// re-export shuffle when the body lands. Stub bodies are minimal -- just
// enough to compile.

#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub enum EdgeKind {
    Calls,
    Implements,
    Extends,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub enum ResolutionMethod {
    SameFile,
    ImportResolved,
    GlobalUnique,
}

/// Pre-resolution output of the edge-find phase (CONTEXT.md lines 39-43).
/// W4 fills the calls_walker that produces these.
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct FoundEdge {
    pub from_file: String,
    pub from_name: String,
    pub to_raw: String,
    pub kind: EdgeKind,
    // Implementation-internal extension fields (NOT part of CONTEXT.md
    // 4-field public contract; see W4 plan FoundEdge interfaces note).
    pub ns_obj: Option<String>,
    pub row: usize,
}

/// Post-resolution Edge with target symbol id + ResolutionMethod (CONTEXT.md
/// lines 54-57). W3 wires this through Snapshot projection.
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct ResolvedEdge {
    pub from_id: i64,
    pub to_id: i64,
    pub kind: EdgeKind,
    pub method: ResolutionMethod,
}
