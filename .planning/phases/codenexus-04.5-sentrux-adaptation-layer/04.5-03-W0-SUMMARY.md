# Phase 04.5-03 W0: Schema Migration Summary

W0 lands the L1 (A3.3) schema change before any sentrux lift or
graph_build.rs rewrite touches storage. After W0, SQLite knows
about `alias_decls` (separate table, composite UNIQUE index `(file,
alias)` created at table-creation time), and `Cmd::Index` refuses to
run silently against a pre-04.5-03 DB. The CHECK constraint on
`edges.kind` STILL accepts `'Imports'` for W0-W2 transition window;
runtime removal rides W3.

## What landed

### Artifacts

- **`core/src/types.rs` (NEW)** -- 4-field `AliasDecl` struct matching
  CONTEXT.md lines 45-52 (`from_file`, `alias`, `target_file`,
  `target_member: Option<String>` where `None` = namespace import).
  `#[allow(dead_code)]` because W0 lands the type for W1+
  consumption; activates in W1 parser sub-crate.

- **`core/src/lib.rs`** -- `pub mod types;` re-export so W1+
  can `use codenexus_core::types::AliasDecl`.

- **`core/src/main.rs`** -- `mod types;` (binary side; bin and lib
  are separate crate roots sharing source per `[lib]+[[bin]]`
  layout) + Cmd::Index pre-flight migration check (G-E).

- **`core/src/storage.rs`** -- (a) `alias_decls` table + composite
  UNIQUE index `alias_decls_file_alias ON alias_decls(file, alias)`
  + secondary `alias_decls_target ON alias_decls(target_file)` index,
  all created inside the same `execute_batch` block as the rest of
  the schema (L1 critical: composite index AT CREATION, not
  retrofitted); (b) `insert_alias_decl` (INSERT OR IGNORE -- idempotent
  on `(file, alias)`); (c) `list_alias_decls` (returns
  `Vec<crate::types::AliasDecl>` ordered by id ASC); (d)
  `clear_alias_decls`; (e) `has_imports_edges` + `has_alias_decls`
  detection helpers for G-E; (f) 5 new unit tests in
  `alias_decls_tests` module.
