## Strategic tension to resolve in 04.5-05

Sentrux ships `metrics/rules/checks.rs` -- TOML-driven rules (`max_cycles`, `max_cc`, `no_god_files`). CodeNexus's stated Software 3.0 north star (PROJECT.md Strategic section) is **semantic ADR indexing** -- MUST/MUST-NOT statements as first-class graph nodes queryable via a new A2A `query_constraints` op.

Three abstraction-level choices:
- **(a)** Lift rules DSL as-is. Rules engine = enforcement layer; ADR indexing = intent layer (Phase 5+).
- **(b)** Skip rules DSL. Build ADR indexing fresh from sentrux's metrics primitives. Forfeit ~600 LoC of ready code to keep strategic vision pure.
- **(c)** Adapt: refactor sentrux's rules engine to consume ADR-derived rules. ADR is input; rules engine is the runtime.

Default recommendation: **(c)**, but only after 04.5-04 lands so DSM/evo integration informs the rule shape. Fall back to (a) if timeline pressure mounts.
