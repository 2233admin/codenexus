### server/internal/supervisor/portscan.go
- `type Lockfile struct { PID int `json:"pid"`; Port int `json:"port"`; StartedAtUnix int64 `json:"started_at_unix"` }`
- `func LockfilePath() string` → ~/.codenexus/port (use os.UserHomeDir + filepath.Join; create parent dir if missing 0700).
- `func ReadLockfile() (*Lockfile, error)` — for CLI subcommands that need to find the running Rust port.
- `func WriteLockfile(lf Lockfile) error` — atomic-ish (write to .tmp + rename).
- `func AcquirePort() (port int, lockfilePath string, err error)`:
    1. If lockfile exists, read it. Check PID alive via os.FindProcess + signal 0 (Windows: use a process-exists helper — `golang.org/x/sys/windows` IsValidProcess equivalent; simplest: try `os.FindProcess(pid)` always succeeds on Windows, so additionally call `process.Signal(syscall.Signal(0))` and check err on POSIX, on Windows fall back to OpenProcess via x/sys/windows or simply use `tasklist` parse — KEEP IT SIMPLE: a small `isProcessAlive(pid int) bool` helper with build-tag separated _unix.go and _windows.go variants is acceptable; if you want zero-deps, just `os.FindProcess` then `proc.Signal(syscall.Signal(0))` works on POSIX, and on Windows assume "if lockfile exists and was modified < 24h ago, treat as alive — caller will fail at port-bind anyway and re-scan").
    2. If alive AND process name verifiable, return its port (no scan, just reuse). If not alive OR name mismatch, unlink lockfile + proceed to scan.
    3. Scan ports 9876..9999: try `net.Listen("tcp", fmt.Sprintf("127.0.0.1:%d", p))`. First successful Listen → close listener immediately, return that port.
    4. All ports busy → return error `fmt.Errorf("all ports 9876..9999 in use")`.
    5. Caller is responsible for writing lockfile after spawn succeeds (so PID is the Rust PID, not Go's).

### server/internal/supervisor/rust.go

- `type State struct { RustAlive bool; RestartCount int; BreakerTripped bool }`

- `func Start(ctx context.Context, cfg Config) (*Supervisor, error)`:
    1. Build *exec.Cmd with cfg.RustBinPath, env: append os.Environ() with CODENEXUS_PORT, CODENEXUS_PORT_LOCKFILE, CODENEXUS_DATA_DIR, CODENEXUS_DEVICE, RUST_LOG, HF_HOME (inherited if user-set, else leave unset).     2. cmd.Start(); on error return wrapped.
    3. Write lockfile {pid: cmd.Process.Pid, port: cfg.RustPort, started_at_unix: now.Unix()} via portscan.WriteLockfile.
    4. Spawn goroutine `s.supervise(ctx)` — implements restart loop + crash-loop breaker.
    5. waitReady: poll http://localhost:<port>/healthz every 500ms; return when 200 OK; timeout after 30s → kill cmd + return error.
    6. Return supervisor.
- `(s *Supervisor) supervise(ctx context.Context)`:
    - Loop until ctx.Done():
        - cmd.Wait() blocks until Rust exits.
        - Lock state: RustAlive=false. If now-lastStableStart >= 5min, reset restartTimes=[].
        - Append now to restartTimes; trim to last 60s.
        - If len(restartTimes) >= 5 → state.BreakerTripped=true; slog.Error fatal; cancel ctx (caller exits non-zero).
        - Else compute backoff: 1s, 2s, 4s, 8s, 16s, then 30s (clamped).
        - Sleep backoff (or until ctx.Done).
        - Re-spawn cmd via same builder. Update lockfile with new PID. Re-await /healthz. lastStableStart=now. RustAlive=true. RestartCount++.
- `(s *Supervisor) State() State` — returns snapshot under mu.
- `(s *Supervisor) Stop() error` — graceful kill: SIGTERM (or os.Process.Kill on Windows since SIGTERM not portable), wait up to 5s, then Kill. Unlink lockfile.
